Лабораторная работа №8

Соответствует лекции: Лекция №6 — Прикладные аспекты применения Claude в бизнесе и образовании

🎯 30 шагов выполнения

1
Откройте Claude.ai и создайте новый диалог.
2
Введите промпт: «Напишите на Python функцию filter_and_transform, которая принимает список целых чисел и возвращает новый список, содержащий квадраты только чётных чисел. Функция должна содержать аннотации типов и docstring в формате Google. Добавьте комментарии к ключевым строкам».
3
Дождитесь ответа и скопируйте сгенерированный код в файл `lab8_filter_transform.py`.
4
Откройте среду разработки Python (IDLE, VS Code или Jupyter Notebook).
5
Создайте новый файл `test_filter.py` и вставьте скопированный код.
6
Добавьте в конец файла тестовый вызов: `print(filter_and_transform([1,2,3,4,5,6]))`.
7
Запустите файл: `python test_filter.py`.
8
Проверьте, что ожидаемый результат: `[4, 16, 36]`.
9
Если есть ошибки, скопируйте их в Claude и попросите исправить.
10
Вернитесь в диалог с Claude.
11
Введите промпт: «Напишите набор unit-тестов с использованием фреймворка unittest для функции filter_and_transform. Покройте как минимум: обычный случай, пустой список, список без чётных чисел, список с отрицательными числами».
12
Скопируйте сгенерированные тесты в файл `test_filter_unittest.py`.
13
Запустите тесты: `python -m unittest test_filter_unittest.py`.
14
Зафиксируйте, сколько тестов прошло успешно, в файл `lab8_test_results.txt`.
15
Вернитесь в диалог с Claude.
16
Введите код с ошибкой: «def find_max(numbers): max_num = 0; for num in numbers: if num > max_num: max_num = num; return max_num».
17
Попросите Claude: «Найдите ошибку в приведённой функции. Объясните, в чём заключается ошибка, и предложите исправленный код».
18
Зафиксируйте ответ Claude в файл `lab8_bug_fix.txt`.
19
Проверьте, обнаружил ли Claude проблему с инициализацией `max_num = 0`.
20
Скопируйте исправленный код и протестируйте на списке `[-5, -2, -8]`.
21
Вернитесь в диалог с Claude.
22
Предоставьте Claude фрагмент кода без комментариев (реализацию бинарного поиска).
23
Попросите: «Сгенерируйте подробную документацию для этого кода: описание функции, аргументы, возвращаемое значение, пример использования, асимптотическую сложность».
24
Скопируйте сгенерированную документацию в файл `lab8_documentation.txt`.
25
Оцените полноту и точность документации.
26
Вернитесь в диалог с Claude.
27
Предложите Claude неоптимальный код (с вложенными циклами, избыточными переменными).
28
Попросите: «Предложите рефакторинг этого кода для улучшения читаемости и производительности. Объясните каждое изменение».
29
Скопируйте исходный и рефакторированный код в файл `lab8_refactoring.txt`.
30
Оформите отчёт с выводами о способности Claude генерировать, тестировать и улучшать код.

  4. Насколько полезна сгенерированная документация?
